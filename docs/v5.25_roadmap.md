## §3 · Pitfall Details

### P1 — `compute_4d_multifactor()` 接受 `fundamentals` 參數

**Problem**: `compute_4d_multifactor()` L170-174 用固定 mock 參數:
```python
pe=25.0, roe=1.5, peg_val=1.2, revenue_growth=0.10
```

11 ticker 全部套同樣 → 無法觸發 cap-zone (PEG=28.72 才會, mock 用 1.2 永遠不觸發)。

**Fix** (Rule 3 精準修改):
```python
def compute_4d_multifactor(
    close: np.ndarray, i: int,
    sma_50_arr, rsi_arr, macd_hist_arr, momentum_20d_arr,
    fundamentals: Optional[Dict[str, float]] = None,  # NEW
    volatility: float = 30.0, var_95: float = -2.5,
    max_dd: float = -20.0, sharpe: float = 1.0, beta: float = 1.1,
) -> Dict[str, float]:
    """... (新增 fundamentals 參數)"""
    fund = fundamentals or {}  # None safe fallback
    fund_score = fund_score_multifactor(
        pe=fund.get("pe", 25.0),
        roe=fund.get("roe", 1.5),
        peg_val=fund.get("peg", 1.2),
        revenue_growth=fund.get("growth", 0.10),
    )
```
