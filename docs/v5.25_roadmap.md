### Root Cause

`scripts/data_sources/__init__.py` **MISSING**。

歷史 (`AUDIT_CHANGELOG.md:410`):
> | `data_sources/__init__.py` | 1 | empty | dead code |

v5.8 dead code sweep 把它刪了,但 `scripts/data_sources/fixture_cache.py:106` 仍在用:
```python
from scripts.data_sources.yfinance_fundamentals import fetch_one
```

這需要 `scripts.data_sources` 是 proper Python package,**必須**有 `__init__.py`。

**Pytest 通過 ≠ CLI 實機通過**: pytest 用 `sys.path` 注入 repo root + `_SCRIPTS_DIR` 路徑 hack,繞過這個 bug。v5.24 P2 cap-zone warning TDD guard 只驗證 **frozen mode** + main() 流程,**沒驗證 live mode CLI 實機**。

### 修復方案 (候選)

#### 方案 A — Touch `scripts/data_sources/__init__.py` (1 commit, 最簡潔)

```bash
touch scripts/data_sources/__init__.py
# 1 行空檔案 → scripts.data_sources 變 proper package
```

**Pros**: Rule 3 精準修改 (1 行),5 分鐘,最低風險
**Cons**: 復活 v5.8 sweep 認為 dead code 的檔案 (但其實 lazy import 仍依賴)

#### 方案 B — 改 `fixture_cache.py` 用 in-script mode import

```python
try:
    from scripts.data_sources.yfinance_fundamentals import fetch_one  # pytest mode
except ImportError:
    from data_sources.yfinance_fundamentals import fetch_one  # CLI mode (scripts/ in path)
```

**Pros**: 不需要 `__init__.py`,符合 v5.8 dead code sweep 結論
**Cons**: 多寫一個 fallback try/except,稍微不乾淨

#### 方案 C — 在 `cross_market_real_yfinance_e2e.py` main 開頭加 `sys.path.insert`

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))  # repo root
```

**Pros**: 統一 import 策略
**Cons**: 改 main() 多處,違反 Rule 3 精準修改
