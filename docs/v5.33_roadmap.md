## 4. (b) audit_v53x_dead_code.py Hardening — 詳細計劃

### 目標

`scripts/audit_v53x_dead_code.py` 加 3 個 fidelity 改善,讓 audit 輸出更精準、更好用。

### 4.1 `--only <category>` CLI flag (D5)

**現狀**: SKILL.md 寫支援 `--only` 但實作沒接。

**實作**:
```python
parser.add_argument(
    "--only",
    choices=["critical", "high", "medium", "low", "dead-code"],
    help="Only show findings of specified category",
)
```

**驗證**: `python scripts/audit_v53x_dead_code.py --iteration v5.32 --only high` 預期只輸出 HIGH section。

### 4.2 HARDCODE-DUP noise-filter (D4)

**現狀**: 89 個 MEDIUM 中 ~66 個是容差 `< 0.005` / `< 0.001` 或 format precision (例 `:.4f` 在 docstring),非真實 magic constant。

**實作**: 在 (c) magic number detector 加 noise rule:
```python
# Skip 容差 (tol/epsilon) literal
if re.search(r"(< |delta=)\s*0\.0\d", line):
    continue
# Skip format precision
if re.search(r":\.\d+f|\.\d+f\}", line):
    continue
```

**預期**: MEDIUM 從 89 → ~23 (真實 cross-file constant)。

### 4.3 DEAD-CODE FastAPI decorator filter (D6)

**現狀**: 8 個 FastAPI route (`@app.get("/cross_market")`) 被報 dead code (找不到 caller),實際是 framework-wired。

**實作**: 在 (d) dead code detector 加 framework filter:
```python
# Skip FastAPI/Flask decorated routes
if re.search(r"@(app|router)\.(get|post|put|delete|websocket)\s*\(", src_line):
    continue
```

**預期**: DEAD-CODE 從 1 真實 (`fetch_all`) 變 0 false-positive noise (但 fetch_all 已刪,所以 D6 主要是防未來新增 route 時誤報)。

### Commit 模板

```
feat(v5.33): audit_v53x_dead_code.py hardening (--only + 2 filters)

- 新增 --only <category> CLI flag (對齊 SKILL.md 描述)
- HARDCODE-DUP noise-filter: 跳過容差 < 0.005 + format precision
- DEAD-CODE framework filter: 跳過 FastAPI/Flask decorated routes

預期: MEDIUM 89 → ~23,DEAD-CODE false-positive -100%

Refs: AUDIT_CHANGELOG.md D4, D5, D6
```
