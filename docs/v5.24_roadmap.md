自動 cap-zone collision 偵測,但**只暴露為 API**,沒有整合進 `cross_market_real_yfinance_e2e.main()`。

## §3 · Pitfall Details

### P1 — `recompute_cross_market_with_cap_warnings()` helper

**Problem**: `live_score_engine.recompute_all_scores_with_cap_warnings()` (v5.23 P5) 只回傳
v5_10 / v5_11_3 / std_quant 三層,**沒整合 cross-market 專屬 quantize 結果**。

**Fix**:
```python
# live_score_engine.py 新增
def recompute_cross_market_with_cap_warnings(
    fundamentals: dict[str, dict],
    v511_mod=None,  # optional, for cross-market signal distribution
) -> dict:
    """v5.24 P1 — Cross-market E2E 整合 cap-zone warning。

    等同 recompute_all_scores() + 額外 cross-market signal distribution
    (per compute_signal_distribution_per_ticker) + cap-zone warnings。

    Returns:
        {
            'scores': <same as recompute_all_scores()>,
            'cap_warnings': <same as recompute_all_scores_with_cap_warnings()>,
            'summary': <merged summary + signal_distribution key>,
        }
    """
```

**TDD 測試** (test_v524_cap_warning_e2e_integration.py):
- 3690.HK fixture + peg=28.72 → `cap_warnings[fund.peg].tickers == ['3690.HK']`
- `summary.warning_by_metric[fund.peg] == 1`
- `summary.total_warnings >= 1`
- 其他 10 ticker (PEG<5) 不觸發 fund.peg warning

### P2 — 整合進 cross_market_real_yfinance_e2e.main()

**Problem**: `main()` L343-356 算 v510/v511 scores, L351 quantize_cross_market,但**沒有
cap-zone warning**,operator 不知道 score 是 cap 還是分布。

**Fix** (Rule 3 精準修改):
```python
# main() L343-356 之後新增 (L357 之後插入,不改既有 print):
# v5.24 P2 — Lesson #49 整合:live mode 跑完自動 cap-zone warning
from data_sources.live_score_engine import recompute_all_scores_with_cap_warnings
_cap_result = recompute_all_scores_with_cap_warnings(fundamentals)
for w in _cap_result["cap_warnings"]:
    logger.warning(
        f"⚠️  Cap-zone collision: {w['metric']} "
        f"({w['n_in_cap_zone']}/{len(fundamentals)} tickers, "
        f"threshold={w['threshold_value']}, by_design={w['is_by_design']}): "
        f"{', '.join(w['tickers'][:5])}"
        + ("..." if len(w["tickers"]) > 5 else "")
    )
```

**整合點選擇**:L357 之前(print 區塊結束,fixtures 寫入之前),**不汙染既有 print**。

### P3 — TDD guard 永久化

**Problem**: 3690.HK fixture PEG=28.72 是 v5.22 P42 的真實 outlier (per `docs/AUDIT_CHANGELOG.md:268`)。
未來若有人誤改 `_FUND_CAP_RULES` 把 `fund.peg` 門檻改低,3690.HK 會 silently 不觸發 warning。

**Fix**: `scripts/tests/test_v524_cap_warning_e2e_integration.py`:
- 用 frozen mode fixtures (PEG=28.72 持久化在 `tickers_fundamentals.json`)
- 驗證 `recompute_all_scores_with_cap_warnings(fundamentals_fixture)` 必觸發 fund.peg warning
- 驗證 cross_market_real_yfinance_e2e.main() (`--mode frozen`) 觸發 logger.warning call
  (用 `caplog` fixture 捕獲 logging)
