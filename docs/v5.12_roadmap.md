## 2. v5.11.3 量化後的剩餘 pitfall（v5.12 候選）

### 2.1 Pitfall #32 — Consensus 引擎「加權平均」假分數獨立同分布

**現狀**（`stock_analysis.py:1290-1306`）：
```python
final_scores = {name: pos["score"] for name, pos in final_positions.items()}
weighted_sum = 0
for role, w in weights.items():
    score = final_scores.get(role, 0)
    contrib = score * w
    weighted_sum += contrib
final_score = min(1.0, max(0.0, weighted_sum))
```

**問題**：
- 6 個 analyst 評分**加權平均**，預設各 analyst 誤差獨立
- 實務上 analyst 共享**同樣的 yfinance data**（PE/ROE/price/news），correlation ≠ 0
- 真正的 consensus 應該用 **variance-aware aggregation**（如 Bayesian model averaging 或 Jensen-Shannon divergence weighted by epistemic uncertainty）
- v5.11.3 修了輸入（multifactor），但**沒修輸出（aggregation）**

**v5.12 提案**：
- P32.A: 量化當前加權平均的 variance 假設偏差（單 ticker vs 6-ticker bootstrap）
- P32.B: 引入 analyst uncertainty 估計（基於 input data quality 與 model disagreement）
- P32.C: 改用 uncertainty-weighted aggregation 取代靜態權重

**量化預期**：
- 當前 final_score 的「信心度」（confidence）是 ad-hoc（LLM 主觀）
- 改為 data-driven confidence（基於 analyst variance）應能更穩定

### 2.2 Pitfall #33 — news_score 仍為 3 值啟發式

**現狀**（`stock_analysis.py:872-895`）：
- `news_score` 根據 news_count 區間給分（0條→0.5, 1-3條→0.55, 4-10條→0.65, >10條→0.7）
- 沒用 v5.11 multifactor pattern

**問題**：
- 仍是 hard threshold + cap 設計
- v5.11 audit 沒碰 news_score（focus 在 4 個 multifactor）
- 違背 v5.11「連續線性、無 cap」精神

**v5.12 提案**：
- P33.A: 重構 `news_score_multifactor(news_count, sentiment_strength, source_diversity)` 為連續線性
- P33.B: 拉 RSS feed 多源化時同步驗證（`get_rss_news` 已有 source list）
- P33.C: pytest 12 條量化（看 news 數量變化時 score 是否線性變化）

### 2.3 Pitfall #34 — sentiment_score 仍有 0.15 hard threshold

**現狀**（`stock_analysis.py:897`）：
```python
sentiment_signal = "positive" if sentiment_result['combined_score'] > 0.15 else "negative" if sentiment_result['combined_score'] < -0.15 else "neutral"
```

**問題**：
- 0.15 硬切（不是連續線性）
- v5.11 精神是「連續函數」，但 sentiment signal 仍是分類

**v5.12 提案**：
- P34.A: 改 `sentiment_score` 為 `tanh(combined_score * k)` 連續映射
- P34.B: signal 仍可保留分類（給 LLM 用），但 score 連續化
- P34.C: pytest 8 條驗證（不同 combined_score 下 score 平滑變化）

### 2.4 Pitfall #35 — 加權權重 hard-coded

**現狀**（`stock_analysis.py` weights dict）：
- weights = {"market": 0.20, "technical": 0.20, "fundamental": 0.20, "risk": 0.15, "sentiment": 0.10, "news": 0.05, "macro": 0.10}
- 寫死，沒根據 ticker profile 動態調整

**問題**：
- AAPL 應該 fundamental 重（高 PE/ROE 重要）
- 0700.HK 應該 technical 重（亞股波動大）
- 不同 ticker 不應用同一組權重

**v5.12 提案**：
- P35.A: 引入 ticker profile（market_cap, sector, region）動態權重
- P35.B: 加權學習 — 從歷史 final_score 對 7 日後實際 return 的 regression 學權重
- P35.C: pytest 10 條 + backtest 量化改善
