## 1. v5.25 Closure 量化發現 (Lesson #52 量化決策)

### v5.25 真實 fundamental 注入後結果

| 指標 | mock GBM | 真實 fund 注入 | 結論 |
|------|----------|----------------|------|
| 3690.HK v5.10 accuracy | 0.5918 (mock seed=42) | 0.5918 (同 mock GBM) | v5.10 純技術,不受 fund 影響 |
| 3690.HK v5.11.3 accuracy | 0.5918 (mock seed=42) | 0.3469 (mock GBM + 真實 fund) | **Δ -0.2449** |
| AAPL v5.11.3 accuracy | 0.5918 (mock) | 0.5918 | 不變 (AAPL fund 健康) |
| 信號分布 (v5.10) | 11 BUY / 473 SELL / 55 HOLD | 同 mock | v5.10 全 SELL |
| 信號分布 (v5.11.3) | 258 BUY / 0 SELL / 281 HOLD | 同 mock | v5.11.3 改分布為 BUY/HOLD |

### 根因分析

`generate_mock_prices(seed=42)` 在 11 ticker 共享同一 close prices array — **v5.10 用技術指標,輸出 473 SELL(幾乎全 SELL)**,**v5.11.3 4D 加權把 SELL 壓到 0,改分布為 BUY/HOLD**。

**真實 cross-market E2E 結果**(per v5.24 closure smoke test):
- 11 ticker 都有真實 close prices (從 yfinance 拉取)
- AAPL 與 3690.HK 波動率差異 ~3x (AAPL vol ~25%, 3690.HK vol ~50%)
- 真實信號分布遠比 mock 多樣(各 ticker 不同)

**Lesson #53 延伸發現**:**mock GBM 隱藏了 v5.11.3 4D 加權的真實表現**。真實 fundamental 注入只解決一半問題(per-ticker fundamental 差異),**close prices 仍是 mock** → 完整 E2E 驗證需要真實 close prices。

## 3. (b) Backtest 真實 close prices 注入 — 詳細計劃

### 目標

`run_cross_market_comparison()` 改用 fixture 內的 `close_prices` 替代 `generate_mock_prices(seed=42)`:

```
fixture: tests/fixtures/tickers_fundamentals.json
  + 新增 key: "close_prices": {ticker: [120 floats], ...}
  - 來源: yfinance 拉取快照 (一次性, 寫入 fixture 後凍結)
  - 大小: 11 ticker × 120 day × ~8 chars/float = ~11KB JSON

backtest: run_cross_market_comparison()
  - 改 close 來源: generate_mock_prices(seed) → fixture.close_prices[ticker]
  - 行為: 每 ticker 用各自的真實 close prices
  - 不變: fund (pe/roe/peg/growth) 注入路徑
```

### P1 — Fixture 擴展 + close prices 注入

**Commit 1 (red light)**: `test(v5.26 P1 red): run_cross_market_comparison 真實 close prices`
- 5+ pytest guards:
  - 1. fixture 包含 `close_prices` key, 11 ticker 全有
  - 2. 每 ticker close prices 陣列長度 = 120 days
  - 3. 任何 price > 0 (sanity check)
  - 4. `run_cross_market_comparison(close_source="real")` 改用 fixture close prices
  - 5. `run_cross_market_comparison(close_source="mock")` 對齊 v5.25 mock 行為 (向後相容)

**Commit 2 (green)**: `feat(v5.26 P1): close_prices 注入 + fixture 擴展`
- 新增 `scripts/snapshot_close_prices.py`:
  - 從 yfinance 拉取 11 ticker × 120 day close prices (one-shot, network 一次)
  - 寫入 `tests/fixtures/tickers_fundamentals.json` 的 `close_prices` key
  - 含 `_meta.snapshot_date` 記錄拉取時間
- `backtest_v511_multifactor.py`:
  - `run_cross_market_comparison()` 加 `close_source: Literal["mock", "real"]` 參數
  - `close_source="mock"` → 沿用 `generate_mock_prices(seed)` (v5.25 行為, 向後相容)
  - `close_source="real"` → 從 fixture 拿 `close_prices[ticker]`

## 5. Risks & Mitigations

| Risk | 影響 | Mitigation |
|------|------|------------|
| yfinance network 失敗 (snapshot 一次性) | snapshot_close_prices.py fail | 寫入 fixture 前 validate, retry 3x, fallback 寫 `_meta.snapshot_failed=True` |
| Fixture 變大 (~11KB) | git diff 變大 | close prices 用 round(4 decimals) 壓縮 |
| v5.11.3 真實 close prices 下意外惡化 | Lesson #54 拒絕 | 量化結果保留為發現, 不視為 regression |
| Ticker 退市 / yfinance 刪除數據 | snapshot 缺 ticker | fallback `generate_mock_prices(seed)` for missing ticker, log warning |
