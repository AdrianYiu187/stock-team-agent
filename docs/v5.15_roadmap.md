## v5.15 候選 pitfall

### P41 — sentiment news_count cap 線性化（低優先級）

**問題**：news_count ≥120 → nc_factor = 0.95 cap
- 真實 cap rate: 0%（罕見）
- 影響：可忽略

**修法**（低優先級，預期量化收益 <0.5pp）：
```python
# v5.12 P34 (current)
if news_count >= 120:
    nc_factor = 0.95
# v5.15 P41 (proposed)
elif news_count < 200:
    nc_factor = 0.95 + 0.04 * (news_count - 120) / 80  # 0.95 → 0.99
else:
    nc_factor = 0.99  # 最終飽和
```

### P42 — news news_count cap 線性化（中優先級）

**問題**：news_count ≥120 → nc_factor = 0.95 cap
- 真實 cap rate: 15.2%
- 影響：中等

**修法**：
```python
# v5.12 P33 (current)
elif news_count < 120:
    nc_factor = 0.95 * news_count / 120
else:
    nc_factor = 0.95  # cap
# v5.15 P42 (proposed)
elif news_count < 200:
    nc_factor = 0.95 + 0.04 * (news_count - 120) / 80  # 0.95 → 0.99
else:
    nc_factor = 0.99  # 最終飽和
```

### P43 — news region_count cap 線性化（**高優先級**）

**問題**：region_count ≥3 → rc_factor = 0.95 cap
- 真實 cap rate: **50.1%**（超半數據飽和）
- 影響：嚴重

**修法**：
```python
# v5.12 P33 (current)
elif region_count < 3:
    rc_factor = 0.30 + 0.65 * region_count / 3
else:
    rc_factor = 0.95  # cap
# v5.15 P43 (proposed)
elif region_count < 5:
    rc_factor = 0.95 + 0.03 * (region_count - 3) / 2  # 0.95 → 0.98
else:
    rc_factor = 0.98  # 最終飽和
```

### P44 — news source_diversity cap 線性化（**高優先級**）

**問題**：source_diversity ≥6 → sd_factor = 0.95 cap
- 真實 cap rate: **58.8%**（過半數據飽和）
- 影響：嚴重

**修法**：
```python
# v5.12 P33 (current)
elif source_diversity < 6:
    sd_factor = 0.30 + 0.65 * (source_diversity - 1) / 5
else:
    sd_factor = 0.95  # cap
# v5.15 P44 (proposed)
elif source_diversity < 12:
    sd_factor = 0.95 + 0.04 * (source_diversity - 6) / 6  # 0.95 → 0.99
else:
    sd_factor = 0.99  # 最終飽和
```
