## v5.14 候選 pitfall（已驗證 scope）

### P37 — market_score_multifactor pos_52wk 線性化

**範圍**：4 個 cap segments (F1-F4) → 線性漸進
**修法建議**：
```python
# v5.10 C23 partial fix: pos 0-20 分段 (1.0/0.95→0.70)
# v5.14 P37: 統一為 [0, 100] 連續線性
if pos_52wk <= 100:
    pos_factor = 1.0 - 0.5 * pos_52wk / 100  # 1.0 (pos=0) → 0.5 (pos=100)
else:
    pos_factor = 0.5
```
**量化預期**：market cap zone 覆蓋率 94.3% → < 5%（極端值保留 cap）

### P38 — market from_high_pct + ytd_return 邊界線性化

**範圍**：F5 (from_high<-60) + F6 (ytd<-100)
**修法建議**：
```python
# from_high_pct: 已有 [-60, +100] 線性，外推到 [-200, -60]
if from_high_pct <= -60:
    # 線性 1.0 (fhigh=-60) → 0.5 (fhigh=-200) — 越深跌風險越高
    dd_factor = 1.0 - 0.5 * (-60 - from_high_pct) / 140
# ytd: 已有 [-100, +200] 線性，外推到 [-200, -100]
if ytd_return <= -100:
    # 線性 0.0 (ytd=-100) → -0.5 → clip 0.0 (ytd=-200 異常崩盤)
    ytd_factor = max(0.0, -0.5 * (-100 - ytd_return) / 100)
```
**量化預期**：market cap zone 94.3% → < 10%

### P39 — tech_score_multifactor 5 cap 線性化

**範圍**：F8 (rsi<5) + F9/F10 (macd ±2) + F11 (ma50<=0) + F12 (mom<-50)
**修法建議**：
- RSI<5: 線性 1.0 → 1.05（但 cap 1.0）— 改成極度超賣加分至 1.0 但允許 0.5 step 變化（其實 rsi=0 vs rsi=5 應該 1.0 vs 0.99）
- macd: 線性 [-2, +2] 已連續；外推到 ±10
- ma50: 保留 fallback（ma50=0 是數據錯誤，給中性 0.5 OK）
- momentum <-50: 線性外推到 -100 (極端崩盤)

### P40 — risk_score_multifactor var_95 + max_dd 正值 cap 線性化

**範圍**：F13 (var>=0) + F14 (dd>=0)
**修法建議**：
- var_95 >= 0：var>=0 表示無下行風險 → 應給 0.85 而非 0.7（更樂觀）
- max_dd >= 0：dd>=0 表示無回撤 → 應給 0.85 而非 0.7

**量化預期**：19% → < 5%
